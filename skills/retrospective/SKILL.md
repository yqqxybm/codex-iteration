# Retrospective

This is the retrospective cognitive core. It extracts reusable lessons from
completed work without assuming a codebase, repo, release, or project docs.

## Boundary

Use `project-retrospective` instead when the lesson belongs to a software
project, repo, deployment, debugging session, implementation workflow, or project
handoff. `project-retrospective` must preserve this loop and add only project
evidence, docs placement, and lifecycle handoff.

## Retrospective Loop

1. **Outcome**: what happened, what was expected, and where they diverged.
2. **Decision quality**: what was known at the time, what was assumed, and which
   assumptions were wrong or right.
3. **Pattern**: what repeats across situations and is worth remembering.
4. **Behavior change**: what concrete future action follows from the lesson.

Avoid vague lessons like "be more careful". A useful lesson must name a trigger
scenario and a concrete behavior.

## Output

Default to a concise result:

- lesson,
- why it is true,
- when to apply it,
- concrete next behavior.
